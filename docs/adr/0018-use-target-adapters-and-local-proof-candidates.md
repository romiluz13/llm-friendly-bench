# Use target adapters and local proof candidates

Before verified Instrumented Agent Runs exist, the product will generate Local Proof Candidates through a small target-adapter seam: one canonical Scenario Fixture becomes database-native MongoDB and Postgres projections, each projection runs the same Order Exception Workflow, and the shared acceptance contract must pass before a Replay Artifact is emitted. This keeps the Lab Console honest and useful while live Codex, Claude Code, and Cursor runs are added later.

Local Proof Candidates may show captured fixture, projection, acceptance, schema, and cost-model evidence, but they must not be labeled verified. Proof Gates prevent promotion to verified until live agent trace, inspected context, retries, code diffs, screenshots, Independent Design Review, and Proof Packet evidence are captured.
